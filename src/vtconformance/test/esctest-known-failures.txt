# --- Intentional gap: X11 Xcms device-dependent colour spaces --------------------------------------
# These assert byte-exact sRGB values produced by libX11's Xcms pipeline for the CIE* / RGBI / TekHVC
# colour specs (e.g. `CIEXYZ:0.5/0.5/0.5` -> `dddd/b5b5/a0a0`, `rgbi:0.5/0.5/0.5` -> `c1c1/bbbb/bbbb`).
# The unequal channels from equal input prove the values come from libX11's built-in per-channel
# phosphor-gamma characterisation tables, not any portable standard -- there is no "X screen" to
# characterise off X11. xterm passes only because it delegates to XParseColor/Xlib; the entire modern
# terminal ecosystem (foot, wezterm, kitty, ghostty, xterm.js, konsole, windows-terminal) declines to
# implement CIE colour spaces. A reference terminal honours the documented portable colour formats
# (rgb:, #hex), not one library's colour-management internals. See docs/internals/vt-conformance.md.
